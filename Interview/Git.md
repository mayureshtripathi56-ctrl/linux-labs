# Git Interview Questions

## Beginner Level

### Q1. What is Git?

Answer:

Git is a Distributed Version Control System (DVCS) that tracks changes made to files over time. It enables developers to save versions, collaborate efficiently, and restore previous versions.

---

### Q2. What is Version Control?

Answer:

Version Control is a system that records changes made to files over time, allowing developers to restore previous versions whenever required.

---

### Q3. What is a Repository?

Answer:

A Repository is a project folder tracked by Git.

---

### Q4. What is GitHub?

Answer:

GitHub is a cloud platform that hosts Git repositories and enables collaboration among developers.

---

### Q5. Difference between Git and GitHub?

Answer:

Git is software that tracks project history.

GitHub is an online platform that stores Git repositories.

---

### Q6. What is a Commit?

Answer:

A Commit is a saved snapshot of a project stored in the local Git repository.

---

### Q7. Difference between Commit and Push?

Answer:

Commit saves changes locally.

Push uploads commits to GitHub.

---

### Q8. Can Git work without internet?

Answer:

Yes.

Git works completely offline.

---

### Q9. Can GitHub work without internet?

Answer:

No.

GitHub requires internet connectivity.

---

### Q10. Why do companies use Git?

Answer:

Companies use Git to collaborate safely, track changes, restore previous versions, and manage software development efficiently.
