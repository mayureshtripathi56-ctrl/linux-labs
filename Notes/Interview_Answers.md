# Interview Answers

## Explain Git.

Git is a Distributed Version Control System that tracks changes in files over time. It enables developers to save versions of their work, restore previous versions, collaborate efficiently, and maintain complete project history.

---

## Explain GitHub.

GitHub is a cloud platform that hosts Git repositories, allowing developers to collaborate, review code, back up projects, and manage software development efficiently.

---

## Explain Git and GitHub to a Non-Technical Person.

Imagine we are writing a book with several editors.

Git is like a register that records every change made to the book—what changed, who changed it, and when it changed.

GitHub is like a secure online locker where both the book and the register are stored safely so that everyone can access the latest version and collaborate without losing previous work.

---

## What is a Commit?

A commit is a saved snapshot of the project stored in the local Git repository.

---

## Difference Between Commit and Push

Commit saves changes locally.

Push uploads committed changes to GitHub.
